# Orbital Debris 3D — Руководство пользователя

**Версия:** v3.0.6  
Браузерная симуляция орбитального мусора. Запустите объект на орбиту, взорвите его и наблюдайте за судьбой фрагментов.

---

## Быстрый старт

1. Откройте `Orbital_Debris_3D_v3_0_6/index.html` в браузере.
2. Установите параметры: высоту орбиты, наклонение, число осколков.
3. Нажмите **▶ Старт** — объект начнёт движение.
4. Нажмите **💥 Взрыв** в любой момент.
5. Наблюдайте; при необходимости ускоряйте время слайдером «Время×» или отматывайте вперёд кнопкой ⏩.

---

## Кнопки управления

| Кнопка | Действие |
|--------|----------|
| **▶ Старт / ⏸ Пауза** | Запуск или остановка симуляции |
| **⟳** | Полный сброс к начальному состоянию |
| **💥 Взрыв** | Немедленный взрыв объекта в текущей позиции |
| **⊡ Вид** | Сброс камеры к виду по умолчанию |
| **⛶ Полный** | Показать всю сцену (bounding box по осколкам) |

| **⊙ Все орбиты** | Показать / скрыть орбитальные кривые всех живых фрагментов |
| **⬡ 2D / 3D** | Переключение видов (2D доступен только при наклонении 0°) |
| **Осколки** | Показать / скрыть индивидуальные осколки |
| **Облако** | Показать / скрыть статистическое облако частиц |

## Параметры запуска и взрыва

### Первая строка параметров

**Распределение скоростей** — закон начальных скоростей фрагментов:

| Режим | Описание |
|-------|----------|
| **Равномерное** | Все фрагменты разлетаются с одинаковой скоростью |
| **Экспоненциальное** | Большинство медленные, единицы — быстрые; реалистичнее для взрывов |
| **Максвелл–Больцман** | Статистическое распределение скоростей, как у молекул газа |

**Наклонение** — угол плоскости орбиты к экватору: 0° / 53° / 70° / 97° (полярная). При наклонении ≠ 0° режим 2D недоступен.

**Время×** — множитель скорости симуляции от 1× до 3000×.

### Вторая строка параметров

| Параметр | Диапазон | По умолчанию | Описание |
|----------|----------|--------------|----------|
| **\|ΔV\| разлёта** | 100–4000 м/с | 500 м/с | Скорость разлёта фрагментов при взрыве |
| **Разгон** | −4000…+4000 м/с | 0 м/с | Добавка к орбитальной скорости при старте; отрицательная — ретроградно |
| **Масса** | 1–100 000 кг | 200 кг | Масса объекта; влияет на размер (→ площадь) и торможение в атмосфере |
| **Высота** | 150–2000 км | 500 км | Высота начальной круговой орбиты |
| **Осколки** | 36–1000 | 200 | Количество индивидуально симулируемых фрагментов |
| **Размер** | 0.5–6 | 2.5 | Визуальный размер осколков на экране |
| **Облако** | 1–30 | 2 | Размер точек статистического облака |
| **🌫 Атмосфера** | вкл/выкл | вкл | Атмосферное торможение |

### Перемотка

1. Введите число в поле рядом с **⏩**.
2. Выберите единицу: **дн. / мес. / лет**.
3. Нажмите **⏩** — прогресс в % на кнопке.
4. Нажмите **✕** для прерывания — симуляция переключится в реальное время 5×.

## HUD — что отображается

### До взрыва

```
t = 0 мин          ← симуляционное время
Виток = 0.00       ← число пройденных витков
Высота орб. = 500 км
V = 7.61 км/с
```

Предупреждения:
- `⚠ объект деорбитировал` — объект вошёл в атмосферу до взрыва.
- `↗ улетел по гиперболе` — скорость превысила вторую космическую.

### После взрыва (при выбранном фрагменте)

```
t = 3 д. 2 ч. 15 мин
Виток = 1.47
Высота орб. = 487 км
V = 7.63 км/с
Апогей   = 612 км
Перигей  = 483 км
Наклонение = 0.3°
Эксцентр.  = 0.0093
```

### Панель статистики (нижний левый угол)

Появляется после взрыва:

| Строка | Значение |
|--------|---------|
| **Осколков на орбите** | Живые фрагменты в зоне наблюдений |
| **За границей наблюдений** | Ушли дальше 400 000 км, но на эллиптической орбите — вернутся |
| **Деорбитировали** | Сошли с орбиты и сгорели в атмосфере |
| **Потеряно в космосе** | Ушли по гиперболической траектории — не вернутся |
| **Макс. апогей** | Наибольший достигнутый апогей среди всех фрагментов |
| **Мин. перигей** | Наименьший перигей — индикатор близости к деорбитации |

## Выбор объектов

- **Клик по объекту** (главный или фрагмент) — выбор; в HUD появляются орбитальные элементы, в 3D — орбитальная кривая (бирюзовая для фрагмента, жёлтая для главного).
- **Клик по пустому месту** — снятие выбора.
- После выбора станет доступна кнопка **◎ Фокус** — камера начнёт следить за объектом.

## Что интересно наблюдать

### Растяжение роя

Сразу после взрыва все фрагменты близко друг к другу. Уже через несколько витков они расходятся вдоль орбиты — фрагменты с большим апогеем движутся медленнее. Через несколько дней рой превращается в кольцо. Статистическое облако показывает этот процесс особенно наглядно.

### Прецессия орбитальных плоскостей

При наклонённых орбитах (53°, 70°, 97°) J2-прецессия RAAN вращает плоскости орбит разных фрагментов с разной скоростью. Через годы кольцо превращается в равномерный тор вокруг Земли — именно так выглядят реальные пояса орбитального мусора.

### Гиперболические траектории

При высоком |ΔV| разлёта часть фрагментов получает скорость выше второй космической (~11.2 км/с относительно Земли). Они уходят по гиперболическим траекториям навсегда — в HUD: `↗ улетел по гиперболе`, апогей обозначается `∞`. В облаке гиперболические частицы окрашены холодным синим цветом.

### Атмосферный сход

Фрагменты с перигеем ниже ~200–300 км быстро тормозятся и деорбитируют. На высоких орбитах (500+ км) этот процесс идёт годами.

### Режим «Все орбиты»

Кнопка **⊙** рисует орбитальные кривые всех живых фрагментов. Хорошо показывает расхождение орбит со временем и прецессию орбитальных плоскостей.

### Влияние распределения скоростей

- **Равномерное:** облако расширяется симметрично, все фрагменты на схожих орбитах.
- **Экспоненциальное:** несколько быстрых фрагментов уходят далеко (вплоть до гиперболы), основная масса остаётся компактно.
- **Максвелл–Больцман:** промежуточный случай, наиболее физически реалистичный.

### Параметр «Разгон»

Положительный разгон повышает начальную орбитальную скорость → эллиптическая орбита с увеличенным апогеем. Отрицательный → снижает перигей вплоть до деорбитации.

## Достоверность модели

### Что соответствует реальной физике

- Орбитальная механика: RK4-интеграция с J2-прецессией и 14-слойной моделью атмосферного торможения.
- Аналитическая модель орбит при перемотке с корректной J2-прецессией RAAN и ω — расползание облака в кольцо/тор соответствует реальной долгосрочной динамике.
- Корректное определение гиперболических траекторий по знаку удельной энергии.
- Баллистический коэффициент, зависящий от массы объекта.
- Качественно верная долгосрочная динамика: растяжение вдоль орбиты, прецессия, атмосферный сход.

### Что упрощено или условно

**Переменная площадь сечения, фиксированная форма.** Все фрагменты предполагаются сферами однородной плотности 200 кг/м³. Реальные обломки имеют разные форму и сечение.

**Изотропный взрыв.** Разлёт равномерно по всем направлениям. Реальный взрыв анизотропен.

**Число индивидуальных фрагментов ограничено.** До 1000 из соображений производительности браузера. Реальный взрыв спутника — тысячи отслеживаемых обломков. Статистическое облако (6000 частиц) компенсирует это визуально.

**Нет взаимодействия между фрагментами.** Столкновения не моделируются (каскад Кесслера внутри облака).

**Земля не вращается физически.** Вращение Земли на экране — визуальный эффект; физика ведётся в инерциальной ECI.

**Граница наблюдений 400 000 км — условная.** За ней физика не считается.

### Итог

Orbital Debris 3D — физически достоверная симуляция орбитальной механики и долгосрочной динамики обломков. Механизм взрыва упрощён; реальное количество фрагментов и их разнообразие значительно больше. Программа позволяет корректно исследовать качественное поведение облака мусора на масштабах от витков до лет.

## Требования

- Браузер с поддержкой WebGL: Chrome 90+, Firefox 88+, Safari 15+, Edge 90+.
- Интернет-соединение для загрузки Three.js r128 с CDN (при первом открытии; затем кэшируется).
- Рекомендуется не менее 4 ГБ RAM при числе фрагментов > 500 и включённом режиме «Все орбиты».
