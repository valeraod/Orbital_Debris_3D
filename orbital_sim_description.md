# Orbital Debris 3D — Техническое описание

**Версия:** v2.10.18  
**Тип:** Одиночный HTML-файл, браузерная симуляция  
**Движок рендеринга:** Three.js r128 (3D WebGL) + Canvas 2D (собственный рендерер)  
**Физика:** Собственная реализация, JS

---

## Назначение

Orbital Debris 3D — интерактивная симуляция орбитального мусора. Пользователь размещает объект на орбите Земли, задаёт параметры взрыва и наблюдает за распространением облака фрагментов: изменением орбит, атмосферным сходом, гиперболическими траекториями и долгосрочной динамикой на масштабах лет. Программа представляет собой чистый исследовательский инструмент без игровых целей и ограничений.

---

## Физическая модель

### Интеграция движения

- **RK4** — четвёртый порядок Рунге–Кутты, без выделения памяти на шаге (reusable `Float64Array(24)` для k1–k4).
- **Адаптивное разбиение шага:** количество подшагов зависит от высоты орбиты — при `h < 200 км` шаг не более 2 с, при `h < 300 км` — 10 с, выше — 60 с.
- Применяется ко всем фрагментам и к главному объекту.

### Силы

| Сила | Применение |
|------|-----------|
| Центральное поле Земли `GM = 3.986e14` | Главный объект, фрагменты |
| J2-прецессия `J2 = 1.08263e-3` | Главный объект, фрагменты |
| Атмосферное торможение | Опционально; 14-слойная модель ATMO_LAYERS (0–1500 км) |

### Баллистический коэффициент

```
BCOEF = m / (Cd × A),  Cd = 2.2,  A = 0.45 м²  (сфера ~75 см)
```

Масса задаётся пользователем; площадь поперечного сечения фиксирована.

### Главный объект

- По умолчанию начинает движение по круговой орбите высотой `HORB` (настраивается).
- Опция **Разгон (ΔV запуска)** — прибавка к орбитальной скорости вдоль вектора скорости при старте; позволяет задать эллиптическую или гиперболическую орбиту.
- Интегрируется RK4 с атмосферным торможением и J2.
- При `r < RE` — деорбитация; при `r > R_OBS = 400 000 км` — выход за границу наблюдений.

### Фрагменты

При взрыве порождается `NFRAG` частиц с изотропным разлётом. Скорость разлёта распределяется по одной из трёх моделей:

| Модель | Описание |
|--------|----------|
| **Равномерное** | Все фрагменты получают одинаковое `|ΔV|` (с небольшим разбросом ±20% при `NFRAG > 36`) |
| **Экспоненциальное** | Большинство медленные, редкие — быстрые; математическое ожидание = `dvMag`; кэп 5× |
| **Максвелл–Больцман** | Сумма трёх нормальных компонент `σ = dvMag/√3`; соответствует тепловому распределению скоростей |

Каждый фрагмент несёт: позицию, скорость, трейл (`TRAIL_LEN = 18`), флаги `alive / beyond / escaped`.

### Граница наблюдений и жизненный цикл фрагментов

- `R_OBS = 4×10⁸ м` (400 000 км) — граница наблюдений.
- Фрагменты, вышедшие за `R_OBS`, остаются «живыми» в физическом смысле, но помечаются `beyond = true` и не рендерятся.
- При выходе за границу определяется тип орбиты: `escaped = (v² / 2 - GM/r ≥ 0)`.
  - `escaped = true` → гиперболическая траектория, улетел в космос.
  - `escaped = false` → эллиптическая орбита с апогеем > 400 000 км, вернётся.
- При `r < RE` → деорбитация, `alive = false`.

## Орбитальные элементы

Функция `orbitalElements3D` вычисляет полный набор элементов из вектора состояния:

- Большая полуось `a`, эксцентриситет `e`, перигей `rp`, апогей `ra`, наклонение `inc`.
- Вектор момента импульса `(hx, hy, hz)`, вектор эксцентриситета `(ex, ey, ez)`.
- Флаг `hyperbolic` — определяется по знаку удельной энергии `ε = v²/2 - GM/r`.

## Рендеринг

### Два режима — переключаются кнопкой «2D / 3D»

#### 3D (Three.js r128)

- Координатное соответствие: физика — ECI Y-up (нестандартно); Three.js — Y-up. Земля вращается визуально (`earthRot += 0.0002` за кадр), физика не учитывает вращение Земли.
- Главный объект — жёлтая сфера с ореолом; масштаб логарифмически зависит от зума.
- Фрагменты — общая геометрия сферы, индивидуальные материалы; выбранный фрагмент масштабируется ×1.8.
- Орбитальная линия главного объекта: живая (сплошная жёлтая) → замороженная (пунктирная жёлтая) после взрыва.
- Орбита выбранного фрагмента — зелёная линия.
- Режим «⊙ Все орбиты» — синие полупрозрачные орбитальные кривые для всех живых фрагментов.
- Гиперболические траектории отображаются корректно (ветвь гиперболы, обрезка по `rMax = 60 × RE`).

#### 2D (Canvas 2D)

- Собственный рендерер, вид сверху на орбитальную плоскость.
- Независимые камера и зум (pan + scroll/pinch).
- Отображаются: Земля, опорная орбита, главный объект, фрагменты, трейлы, орбиты.
- При зуме «близко» показывает `zoom ×` в HUD.

### Ортографическая камера в 3D

В 3D режиме также доступна ортографическая проекция: `OrthographicCamera` с зумом, синхронизированным с состоянием камеры.

## Интерфейс

### HUD (верхний левый угол)

```
t = 0 мин             ← симуляционное время
Виток = 0.00          ← витки главного объекта / выбранного фрагмента
Высота орб. = 500 км  ← текущая высота
V = 7.61 км/с         ← скорость
```

При выборе объекта дополнительно:
```
Апогей   = 500 км
Перигей  = 500 км
Наклонение = 0.0°
Эксцентр.  = 0.0000
```

Предупреждения: `⚠ объект деорбитировал`, `↗ улетел по гиперболе`.

### Панель статистики (нижний левый угол)

Появляется после взрыва:
- Осколков на орбите / всего (с прогресс-баром).
- За границей наблюдений (вышли за 400 000 км, но живые).
- Деорбитировали (упали).
- Потеряно в космосе (гиперболическая траектория).
- Макс. апогей / Мин. перигей среди живых фрагментов.

### Панель управления

**Первая строка:** Старт / Сброс / Взрыв / Вид / Полный / Фокус / Все орбиты / 2D-3D / Распределение скоростей / Скорость симуляции.

**Вторая строка:** |ΔV| разлёта / Разгон / Масса / Высота / Осколки / Размер точек / Атмосфера / Перемотка.

### Выбор объектов (клик)

- Клик по главному объекту — выбор, показ орбитальных элементов в HUD.
- Клик по фрагменту — выбор, показ элементов и орбиты фрагмента.
- Клик по пустому месту — снятие выбора.
- **Фокус** — режим слежения камеры за выбранным объектом.

## Скорость симуляции и перемотка

- Слайдер «Время×»: линейная шкала 1–3000×.
- **Перемотка:** фиксированный шаг 30 с, пакетами с `await setTimeout(0)` для обновления UI; прогресс в % на кнопке.
- В режиме fast-forward трейлы не обновляются (`BASE_SUB = 2` вместо 6 для скорости).
