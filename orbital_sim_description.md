## Назначение

Orbital Debris 3D — интерактивная симуляция орбитального мусора. Пользователь размещает объект на орбите Земли, задаёт параметры взрыва и наблюдает за распространением облака фрагментов: изменением орбит, прецессией плоскостей, атмосферным сходом, гиперболическими траекториями и долгосрочной динамикой на масштабах лет. Чистый исследовательский инструмент без игровых целей.

## Физическая модель

### Интеграция движения

- **RK4** — четвёртый порядок Рунге–Кутты, без выделения памяти на шаге (reusable `Float64Array(24)` для k1–k4).
- **Адаптивное разбиение шага:** количество подшагов зависит от высоты орбиты — при `h < 200 км` шаг не более 2 с, при `h < 300 км` — 10 с, выше — 60 с.
- Применяется к главному объекту и ко всем фрагментам (в реальном времени).

### Силы

| Сила | Применение |
|------|-----------|
| Центральное поле Земли `GM = 3.986e14` | Главный объект, фрагменты |
| J2-прецессия `J2 = 1.08263e-3` | Главный объект, фрагменты |
| Атмосферное торможение | Опционально; 14-слойная модель ATMO_LAYERS (0–1500 км) |

### Баллистический коэффициент

```
BCOEF = m / (Cd × A),  Cd = 2.2
```

Площадь поперечного сечения `A` зависит от массы: предполагается сфера однородной плотности 200 кг/м³ (функция `massToBC`). При малых массах это даёт меньшее сечение и более медленное торможение, при больших — наоборот.

### Главный объект

- По умолчанию начинает движение по круговой орбите высотой `HORB`.
- **Наклонение** задаётся выпадающим списком (0° / 53° / 70° / 97°).
- **Разгон (ΔV запуска)** — прибавка к орбитальной скорости вдоль вектора скорости при старте; позволяет получить эллиптическую или гиперболическую орбиту.
- Интегрируется RK4 с J2 и атмосферным торможением.
- При `r < RE` — деорбитация; при `r > R_OBS = 400 000 км` — выход за границу наблюдений.

### Фрагменты

При взрыве порождается `NFRAG` частиц с изотропным разлётом. Скорость разлёта — одна из трёх моделей:

| Модель | Описание |
|--------|----------|
| **Равномерное** | Все фрагменты получают одинаковое `|ΔV|` (с разбросом ±20% при `NFRAG > 36`) |
| **Экспоненциальное** | Большинство медленные, редкие — быстрые; математическое ожидание = `dvMag`; кэп 5× |
| **Максвелл–Больцман** | Сумма трёх нормальных компонент `σ = dvMag/√3` |

Каждый фрагмент несёт: позицию, скорость, трейл (`TRAIL_LEN = 18`), флаги `alive / beyond / escaped`. При создании также вычисляются орбитальные элементы (`orb`), используемые при перемотке времени.

### Аналитическая модель орбит (перемотка)

Для каждого эллиптического фрагмента при перемотке (`⏩`) позиция вычисляется аналитически через `orbitPosAtTime`:

1. Решение уравнения Кеплера (метод Ньютона, `solveKepler`).
2. J2-прецессия RAAN (Ω) и аргумента перигея (ω) за прошедшее время.
3. Упрощённое атмосферное торможение через плотность у перигея.
4. Матрица Гаусса → ECI-координаты.

Это позволяет мгновенно перемотать симуляцию на годы вперёд с корректной картиной расползания облака, не тратя CPU на тысячи шагов RK4.

### Граница наблюдений и жизненный цикл фрагментов

- `R_OBS = 4×10⁸ м` (400 000 км).
- Фрагменты за `R_OBS` с эллиптической орбитой помечаются `beyond = true` — они ещё вернутся.
- Фрагменты за `R_OBS` с `v² / 2 - GM/r ≥ 0` → `escaped = true` (гиперболическая, не вернётся).
- При `r < RE` → деорбитация, `alive = false`.

## Орбитальные элементы

Функция `orbitalElements3D` вычисляет полный набор из вектора состояния:

- Большая полуось `a`, эксцентриситет `e`, перигей `rp`, апогей `ra`, наклонение `inc`.
- RAAN (Ω), аргумент перигея (ω), средняя аномалия `M0`, истинная аномалия `ν`.
- Вектор момента импульса `(hx, hy, hz)`, вектор эксцентриситета `(ex, ey, ez)`.
- Флаг `hyperbolic` — по знаку удельной энергии `ε = v²/2 - GM/r`.

## Статистическое облако осколков (density_cloud.js)

Параллельно с индивидуальными осколками существует слой из **6000 статистических частиц**, каждая со своими орбитальными элементами. Цель — показать форму распределения мусора (растяжение в кольцо/тор) даже при небольшом числе индивидуальных осколков.

- При взрыве (`dcSpawn`) для каждой частицы вычисляются орбитальные элементы; гиперболические и эллиптические ветки обрабатываются отдельно.
- Позиции обновляются аналитически (`_dcFillArrays`): J2-прецессия, упрощённое торможение.
- **Цвет по энергии и эксцентриситету:** эллиптические — тёплый градиент (красный → жёлтый); гиперболические — холодный (жёлтый → синий).
- При `dcStep()` обновляется каждые 3 кадра (реальное время); при перемотке — `dcStepAnalytical(t)`.

## Рендеринг

### Два режима — переключаются кнопкой «⬡ 2D / 3D»

#### 3D (Three.js r128)

- Координатное соглашение: физика и рендеринг — ECI Z-up; Three.js — Y-up. Координаты передаются напрямую (`mesh.position.set(x, y, z)`).
- Земля вращается визуально (`earthRot += 0.0002` за кадр); физика ведётся в инерциальной ECI без учёта вращения.
- Главный объект — жёлтая сфера с ореолом.
- Орбита главного объекта: живая (сплошная жёлтая) → замороженная (пунктирная) после взрыва.
- Фрагменты — общая геометрия сферы, индивидуальные материалы; выбранный масштабируется ×1.8.
- Орбита выбранного фрагмента — бирюзовая линия (`#00ffb4`).
- Режим «⊙ Все орбиты» — синие полупрозрачные орбитальные кривые всех живых фрагментов.
- Гиперболические траектории отображаются корректно; обрезка по `rMax = 60 × RE`.
- Также доступна ортографическая камера (`OrthographicCamera`).

#### 2D (Canvas 2D)

- Собственный рендерер, вид сверху на орбитальную плоскость (только при наклонении 0°).
- Независимые панорамирование и зум (drag + scroll/pinch), диапазон зума 0.04–20×.
- Отображаются: Земля, опорная орбита, главный объект, фрагменты, трейлы, орбиты.
- При зуме «близко» показывает `zoom ×` в HUD.

## Интерфейс

### HUD (верхний левый угол)

```
t = 0 мин             ← симуляционное время
Виток = 0.00          ← витки главного объекта / выбранного фрагмента
Высота орб. = 500 км
V = 7.61 км/с
```

При выборе объекта дополнительно:
```
Апогей   = 500 км
Перигей  = 500 км
Наклонение = 0.0°
Эксцентр.  = 0.0000
```

Предупреждения: `⚠ объект деорбитировал`, `↗ улетел по гиперболе`.

### Панель статистики (нижний левый угол)

Появляется после взрыва:
- Осколков на орбите / всего (с прогресс-баром).
- За границей наблюдений.
- Деорбитировали.
- Потеряно в космосе.
- Макс. апогей / Мин. перигей среди живых фрагментов.

### Панель управления

**Первая строка:** Старт / Сброс / Взрыв / Вид / Полный / Фокус / Все орбиты / 2D-3D / чекбоксы «Осколки» и «Облако» / Распределение скоростей / Наклонение / Время×.

**Вторая строка:** |ΔV| разлёта / Разгон / Масса / Высота / Осколки / Размер (осколков) / Облако (размер облака) / Атмосфера / Перемотка (поле + единицы + ⏩ + ✕).

## Скорость симуляции и перемотка

- Слайдер «Время×»: 1–3000×.
- **Перемотка ⏩:** аналитическая (через уравнение Кеплера), пакетами по 30 000 с с `await setTimeout(0)` для обновления UI; прогресс в % на кнопке.
- Кнопка **✕** прерывает перемотку и переключает симуляцию в реальное время на 5×.
- В режиме fast-forward трейлы не обновляются.
